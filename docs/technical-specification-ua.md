# Технічна специфікація

**Проєкт:** Інтерактивна карта покриття фіксованим широкосмуговим доступом та підключеності соціальних установ в Україні

**Призначення документа:** пояснити, які технології застосовано в проєкті та з якою метою.

---

## 1. Загальний підхід

Систему побудовано **виключно на відкритому програмному забезпеченні** (open-source), без ліцензійних платежів і без залежності від комерційних постачальників. Це принципове рішення, яке відповідає публічному характеру проєкту, знижує вартість володіння та забезпечує довгострокову стійкість рішення.

Архітектурно систему розділено на дві частини:

- **Аналітичний контур (офлайн)** — обробляє великі масиви геопросторових даних (мільйони об'єктів) на робочій станції.
- **Публічний вебсервіс (онлайн)** — легкий, економний застосунок, що працює на невеликому сервері та обслуговує кінцевих користувачів.

Такий поділ дозволяє важкі обчислення виконувати одноразово, а публічний сервіс тримати компактним, швидким і недорогим в утриманні.

---

## 2. Технологічний стек за призначенням

| Компонент | Технологія | Призначення |
|---|---|---|
| **База геопросторових даних** | PostgreSQL + PostGIS | Центральне сховище всіх даних. Виконує просторове зіставлення соціальних закладів із будівлями та адміністративними одиницями. |
| **Конвеєр обробки даних (ETL)** | Python (GDAL, geopandas) | Завантаження та очищення вихідних даних (шейп-файли забудови, реєстр соціальних закладів із даними про платежі за доступ до Інтернету), приведення їх до єдиного формату та геоприв'язка. |
| **Програмний інтерфейс (API)** | Python + FastAPI | Надає вебзастосунку дані про заклади та їх підключеність у стандартизованому форматі. |
| **Картографічний вебзастосунок** | React + TypeScript + MapLibre GL | Інтерактивна карта в браузері: відображення закладів, фільтри, картки з деталями. |
| **Картографічні дані** | PMTiles + GeoJSON | Ефективна доставка карти будівель та адміністративних меж у браузер без навантаження на сервер. |
| **Розгортання та інфраструктура** | Docker, Caddy | Стандартизоване, відтворюване розгортання на сервері з автоматичним захищеним з'єднанням (HTTPS). |

---

## 3. Обґрунтування ключових рішень

**PostgreSQL + PostGIS** — промисловий стандарт для роботи з геопросторовими даними. Дозволяє опрацьовувати мільйони об'єктів забудови та виконувати просторове зіставлення (визначення будівлі, у якій розташований заклад) із високою продуктивністю. Повністю відкритий, без ліцензійних обмежень.

**Python** для обробки даних — має найзріліший набір інструментів для геопросторового аналізу. Використання однієї мови і для підготовки даних, і для програмного інтерфейсу зменшує ризик помилок і спрощує супровід.

**MapLibre GL** для карти — сучасний відкритий рушій, що забезпечує якісне візуальне відображення без прив'язки до платних сервісів (на відміну від комерційних аналогів, які потребують платних ключів доступу).

**PMTiles** для карти будівель — технологія, що дозволяє обслуговувати великі обсяги картографічних даних як звичайні статичні файли. Це забезпечує роботу з великою кількістю одночасних користувачів практично без витрат на серверні потужності.

**Docker + Caddy** для розгортання — гарантують, що застосунок однаково працює під час розробки та на робочому сервері, з автоматичним налаштуванням шифрування з'єднання. Розгортання й оновлення виконуються стандартизовано.

---

## 4. Вимоги до інфраструктури

Завдяки економній архітектурі публічний сервіс не потребує потужного обладнання. Орієнтовна конфігурація сервера для поточного етапу:

- **2 віртуальні процесори, 4 ГБ оперативної пам'яті, 40 ГБ дискового простору.**

Уся важка обробка даних відбувається окремо, тому сервіс залишається легким і масштабується за потреби зі збільшенням обсягу даних.
